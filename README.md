# retreat_update

Telegram-бот для заявок на `Wellness Camp` и `Family Camp`.

Дополнительно бот отправляет менеджерам новый контакт, когда пользователь впервые нажимает `/start`.

## Что нужно для запуска

- `TELEGRAM_TOKEN` — токен бота из BotFather
- `MANAGER_CHAT_IDS` — личные `chat_id` менеджеров через запятую
- `MANAGER_GROUP_ID` — необязательный legacy fallback для группового чата

Файл `.env.example` показывает формат переменных.

## Почему бот не работал

- Процесс запускался как `web` через `Procfile`, но сам бот работал только через Telegram polling и не поднимал HTTP-порт.
- Для большинства web-деплоев это означает провал health check и бесконечные перезапуски.
- В `camp_bot.py` также был публично засвечен реальный токен бота. Его нужно срочно перевыпустить в BotFather и обновить в переменных окружения.

## Что уже исправлено

- Добавлен простой healthcheck-сервер на `PORT`, чтобы `web`-деплой не падал.
- Добавлена явная проверка обязательных env-переменных с понятной ошибкой в логах.
- Убран токен из исходников и добавлен `.env.example`.

## Что проверить в деплое

1. В настройках сервиса заданы `TELEGRAM_TOKEN` и `MANAGER_CHAT_IDS`.
2. Каждый менеджер хотя бы один раз открыл бота и нажал `/start`, иначе Telegram не даст боту написать ему в личку.
3. Используется новый токен после перевыпуска в BotFather.
4. Сервис действительно деплоит актуальный `main` из этого репозитория.
